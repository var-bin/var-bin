Некоторые проблемы CSS

  • В таблицах стилей очень много повторений
  • Цвета, шрифты, часто используемые группы свойств и т.п.
  • Никакого объектно-ориентированного подхода

Преимущества использования препроцессоров

  • Переменные
  • Повторное использование кода (DRY)
  • Возможность дробить большие файлы стилей на маленькие независимые компоненты
  • Скорость написания кода возрастает в несколько раз

Недостатки использования препроцессоров

  • Вся "магия", которую умеют препроцессоры недоступна после компиляции
  • Непонимание конечного результата
  • На больших проектах компиляция стилей занимает много времени
  • Перейти на "чистый" CSS трудозатратно

Практика

  1. Установите расширение для своего редактора:
    • Sublime Text
    • VS Code
  2. Создайте файл styles.scss
  3. Создайте несколько переменных и примените их в стилях
  4. Скомпилируйте стили
  5. Проверьте итоговый результат

Практика

  1. Создайте навигационное меню для сайта из пяти пунктов. За пример, можно взять меню из своего блога
  2. Примените стили к этому меню используя вложенность
  3. Добавьте несколько многострочных комментариев
  4. Добавьте несколько однострочных комментариев
  5. Используйте переменные
  6. Скомпилируйте стили
  7. Проверьте итоговый результат

Что умеет Sass. Partials

  • Имена таких файлов начинаются с нижнего подчеркивания _
  • Такие файлы используются вместе с @import
  • Пример имени: _partial.scss
  • Пример подключения: @import "partial";

Что умеет Sass. Импорты и Partials

Практика

  1. Создайте файл normalize.scss. Пропишите в него стили
  2. Создайте файл _variables.scss. Пропишите в него переменные
  3. Подключите эти оба файла в styles.scss
  4. Используйте переменные для задания стилей для body
  5. Скомпилируйте стили
  6. Проверьте итоговый результат

Что умеет Sass. Примеси (Миксины)

Практика

  1. Создайте файл _mixins.scss
  2. Добавьте в него несколько миксинов. Например: box-shadow; border-radius. Один, два придумайте сами
  3. Подключите _mixins.scss в styles.scss
  4. Используйте миксины
  5. Скомпилируйте стили
  6. Проверьте итоговый результат

Можете доавить что-то свое, что мы не успели рассмотреть. Сравните результат, который получился с тем, где вы не использовали Sass.
