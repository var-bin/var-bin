Отличие фреймворка от библиотеки

Зачем нужны фреймворки

Плюсы:

  • Решает типичные задачи наилучшим способом
  • При написании фреймворка использовались best practices
  • Кодовая база написана такими же разработчиками как и вы
  • Кроссбраузерность/Кроссплатформенность
  • Тесты

Зачем нужны фреймворки

Недостатки:

  • Вы хотели только банан, а тяните к себе на проект и обезьяну и все джунгли
  • Должны следовать требованиям в документации
  • Переход на новую версию фреймворка может быть трудозатратным, а иногда и невозможным

Reset.css vs. Normalize.css

Reset.css vs. Normalize.css

Normalize.css

  • Сохраняет полезные значения по умолчанию, в отличие от многих сбросов CSS.
  • Нормализует стили для широкого диапазона элементов.
  • Исправляет ошибки и общие несовместимости браузера.
  • Подробно документированный код
Normalize.css

Reset.css vs. Normalize.css

Практика

Подключите к своему блогу сначала Reset CSS и посмотрите как изменилось отображение вашего блога. После этого удалите Reset CSS и подключите Normalize.css. Проверьте результат после подключения Normalize.css.

SVG Path - <path/>

  • d - содержит параметры пути (координаты).

Обозначения:

  • M - "moveto" команда, которая определяет новую точку.
  • L - "lineto" команда, которая "рисует" прямую линию с текущей точки в новую точку.
  • H - горизонтальная "lineto" команда, которая "рисует" прямую линию с текущей точки в новую точку.

SVG Path - <path/>

  • V - вертикальная "lineto" команда, которая "рисует" прямую линию с текущей точки в новую точку.
  • ...
  • Z - "closepath" завершает текущий подпуть и вызывает автоматическую прямую линию, которая будет нарисована от текущей точки до начальной точки текущего подпути.

Практика

SVG. Пишем текст. text

      <svg>
        <text x="15%" y="60%" font-family="Arial" font-size="55"
        fill="red">
          Hello, out there!
        </text>
      </svg>
  • x - Если одно значение, то задает координаты по оси X для первого глифа. Если значения указаны через запятую или через пробел, то каждое значение задает позицию для каждого отдельно взятого глифа по оси X.

SVG. Пишем текст. text

  • y - Если одно значение, то задает координаты по оси Y для первого глифа. Если значения указаны через запятую или через пробел, то каждое значение задает позицию для каждого отдельно взятого глифа по оси Y.
  • rotate - позволяет "вращать" текст. Если значения указаны через запятую или через пробел, то каждое значение задает "вращение" для каждого отдельно взятого глифа.
