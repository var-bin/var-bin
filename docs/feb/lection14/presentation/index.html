npm - Что? Где? Когда?

Как установить npm

  1. Cкачать и установить Nodejs
  2. Cкачать и установить ConEmu. Этот пункт не обязательный. Можно пользоваться и стандартной консолью Windows, но она жутко неудобная.
  3. Запустить ConEmu и выполнить следующие команды: 
      node -v // выведет версию nodejs
      npm -v // выведет версию npm
      sass -v // выведет версию sass
Если первые две команды ничего не вывели, у вас проблемы с установкой

Базовые команды CLI

      // сменить директорию
      cd <path>

      pwd

Базовые команды npm

      // инициализация проекта
      npm init
      // установить пакет
      npm install <package-name>
      // установить и сохранить пакет в "dependencies"
      npm install -S <package-name>
      // установить и сохранить пакет в "devDependencies"
      npm install -D <package-name>

Базовые команды npm

      // установить пакет глобально
      npm install -g <package-name>
      // удалить пакет
      npm uninstall <package-name>
      // удалить пакет и убрать зависимость в "dependencies"
      npm uninstall -S <package-name>
      // удалить пакет и убрать зависимость в "devDependencies"
      npm uninstall -D <package-name>

Базовые команды npm

"dependencies" - эти пакеты подключаются вашим приложением для нормальной работы у конечного пользователя (production).

"devDependencies" - эти пакеты нужны только при разработке и тестировании приложения.

Базовые команды cli и npm

Практика

  1. Перейдите в директорию (если ее нет - создайте) со своим проектом
  2. Создайте файл styles.scss
  3. Добавьте в него стили, миксины, переменные и т.д.
  4. Скомпилируйте данный файл при помощи cli: 
      sass [options] [INPUT] [OUTPUT]
      sass styles.scss styles.css

Как работают постпроцессоры

Примеры постпроцессоров

  • CSSO – минификация css
  • CSSComb – изменение порядка выбора правил
  • cssrb – библиотека для замены ссылок на ресурсы в css файлах
  • Autoprefixer – автоматическое добавление вендорных префиксов

Примеры постпроцессоров

Практика

  1. Установите CSSComb
  2. Создайте .csscomb.json через специальный сервис
  3. Поместите .csscomb.json в директорию с проектом
  4. Выполните следующую команду: csscomb styles.css
  5. Проверьте результат

Постпроцессоры

  1. Rework:
    • первый
    • проще, меньше
  2. PostCSS:
    • более умный парсер
    • лучше поддержка карт кода
    • сохраняет форматирование
    • удобнее API

PostCSS. Используем в качестве CLI инструмента

  1. Установить глобально postcss-cli
  2. Установить глобально autoprefixer
  3. Пример использования вместе с autoprefixer:
      postcss *.css --use autoprefixer -d build/

PostCSS. Используем в качестве CLI инструмента

postcss.config.js - настройки для PostCSS

      module.exports = {
        map: false,
        plugins: [
          require("autoprefixer")()
        ]
      }

PostCSS. Используем в качестве CLI инструмента

Практика

  1. Установить глобально postcss-cli
  2. Создайте postcss.config.js
  3. Добавьте autoprefixer в секцию plugins postcss.config.js
  4. Разместите postcss.config.js в директории с проектом
  5. Создайте styles.css
  6. Добавьте в него стили, которые плохо поддерживаются без вендорных префиксов. Пример: dispay: flex;

PostCSS. Используем в качестве CLI инструмента

Практика

  1. Выполните следующую команду:
      postcss styles.css -c postcss.config.js -d build/

Если все сделали верно, в файл styles.css будут автоматически добавлены вендорные префиксы.

PostCSS. Используем в качестве CLI инструмента

Другие полезные команды

      // Multiple files
      postcss examples/ex1/*.css -c postcss.config.js -r
      postcss examples/**/*.css -c postcss.config.js -r
      // Watch mode
      postcss examples/**/*.css -c postcss.config.js -r -w

PostCSS. Используем в качестве CLI инструмента
